# 08 · Install / Uninstall 脚本

## 定位

两个 bash 脚本（`scripts/` 目录下），各自独立：

- **`scripts/install.sh`**：把一个**生成的** wrapper 写进 `~/.local/bin/llmw`，并按需往 shell
  rc 注册 `PATH`。
- **`scripts/uninstall.sh`**：install 的精确逆操作——删 wrapper、删 `PATH` marker 块。

**全程不碰 pip / pipx / venv**，不安装 `llmw/` 包。依赖模型见下「依赖」一节。

> 这是 Phase 2 的产物（见 `00-overview.md` 的 Phase 边界与 I-4）。

## 关键不变量

延续 `00-overview.md` 的 I-N 约定：

### I-5：install/uninstall 只动两类产物

- `~/.local/bin/llmw`（生成的 wrapper 文件）
- shell rc 里的**一个** marker 块（仅当 `~/.local/bin` 原本不在 `PATH`）

**不碰**：`llmw/` 包本身、克隆下来的仓库目录、用户 workspace 数据、Python 环境、
`claude` CLI。uninstall 不提供任何深度清理 flag（`--purge` / `--remove-repo` 等
均不做）。

### I-6：install 在仓库根直接可跑

- 用户克隆仓库后，在仓库根执行 `./scripts/install.sh`。
- 脚本以**自身所在目录的上一级**定位仓库根（脚本在 `scripts/`，仓库根是其父目录），不需要
  环境变量；用 `repo_root="$(cd "$(dirname "$0")/.." && pwd)"` 求绝对路径（不依赖
  `readlink -f`，兼容 macOS bash 3.2）。
- 生成的 wrapper 内嵌该仓库绝对路径。

### I-7：wrapper 自包含

- wrapper 内嵌仓库绝对路径，通过 `PYTHONPATH` 让 `python3 -m llmw` 找到包，**不依赖**
  运行时 cwd（从任意目录跑都能找到包）。
- 用 `PYTHONPATH` 而非 `cd`，避免改变 cwd 影响用户传入的相对路径参数。
- 不 symlink 到 `bin/llmw`——原文件内容不含 `PYTHONPATH`，从别处跑会找不到包。

## 依赖

| 依赖 | 是否需要 | 说明 |
| --- | --- | --- |
| `python3 ≥ 3.7` | 需要 | 已是 CLI 硬前提；wrapper 本身也调 `python3` |
| `tomli` | **仅 Python < 3.11** | 3.11+ 走 stdlib `tomllib`，零第三方依赖（见 `llmw/_compat.py`） |
| 仓库 + `my_SKILL` submodule | 需要 | 克隆时 `git submodule update --init --recursive`；install 仅做存在性提示，不自动 `git` 操作 |
| `claude` CLI | 仅 `enter` 命令 | 外部工具，不归 install 管 |

install 脚本本身：纯 bash + 系统自带 `python3`，**无第三方依赖**。

## 组件

### ① `scripts/install.sh` + `scripts/uninstall.sh`

两个独立、各自自包含的脚本。**共享契约**是 marker 块的起止行字符串（见下）——
uninstall 只按这两行定位并整段删除，**不依赖** install 当时写了哪个 rc 文件。rc 选择用
同一套 `$SHELL` 启发式各自重推导，因此 uninstall 能独立找到 install 写入的文件（只要用户
shell 未变）。漂移风险仅限这两行字符串，文档与测试固定该字符串即可消除。

调用形式：

```bash
./scripts/install.sh            # 安装
./scripts/install.sh -h|--help  # 用法
./scripts/uninstall.sh          # 卸载
./scripts/uninstall.sh -h|--help
```

### ② 生成的 wrapper（install 时写出）

模板（`<REPO_ABS>` 在 install 时替换为仓库绝对路径）：

```bash
#!/usr/bin/env bash
# llmw launcher — generated by install.sh. Repo: <REPO_ABS>
# 仓库被挪走会失效；重跑 ./scripts/install.sh 即可修复。
export PYTHONPATH="<REPO_ABS>:$PYTHONPATH"
exec python3 -m llmw "$@"
```

### marker 块（写入 rc 的内容）

幂等形式（source 多次也不会重复前缀 `PATH`），POSIX 兼容（zsh / bash / sh 都能跑）：

```sh
# >>> llmw (managed by install.sh) >>>
case ":$PATH:" in
  *":$HOME/.local/bin:"*) ;;
  *) PATH="$HOME/.local/bin:$PATH"; export PATH ;;
esac
# <<< llmw <<<
```

行首 `# >>> llmw (managed by install.sh) >>>` 与 `# <<< llmw <<<` 是 uninstall 的删除
边界。uninstall 按这两行整段删除（含边界行），**块外内容一字不动**。

## install 流程

1. **定位仓库根**：`repo_root="$(cd "$(dirname "$0")/.." && pwd)"`（脚本在 `scripts/`，仓库根是其上一级）。
2. **检测 `python3`**：`command -v python3` 失败 → 非 0 退出并提示。
   - 解析版本；`< 3.11` 时**提示**「可能需要 `pip install tomli`」，不自动安装。
3. **建目录**：`mkdir -p ~/.local/bin`；失败 → 非 0 退出。
4. **生成 wrapper**：把 `<REPO_ABS>` 替换进模板，写到 `~/.local/bin/llmw`，
   `chmod +x`。（reinstall 直接覆盖。）
5. **按需注册 PATH**：
   - 用 `case ":$PATH:" in *":$HOME/.local/bin:"*)` 检测 `~/.local/bin` 是否已在当前
     `PATH`。
   - **已在** → 跳过 rc 改动（uninstall 时无 rc 可清）。
   - **不在** → 按「rc 选择」选出目标 rc 文件；若该文件已有 marker 起始行 → 视为已装、
     跳过（幂等）；否则追加 marker 块。rc 文件不存在则创建。
6. **验证**：执行 `~/.local/bin/llmw --help`，期望 exit 0。失败 → 回滚（删 wrapper、删
   marker 块）并报错。
7. **submodule 提示**：若 `my_SKILL/llm-wiki-management/scripts/setup_wiki.py` 不存在，
   提示跑 `git submodule update --init --recursive`（仅提示，不自动执行）。
8. **输出**：打印安装结果；若改了 rc，提示 `source <rc>` 或重开终端。

### rc 选择（跨平台启发式）

`$SHELL` 取末段判断：

| Shell | 目标 rc | 说明 |
| --- | --- | --- |
| `zsh` | `~/.zshrc` | macOS 默认 |
| `bash` | macOS（`uname` = `Darwin`）→ `~/.bash_profile`；否则 `~/.bashrc` | mac bash 为 login shell；Ubuntu 终端多为非登录，走 `.bashrc`（其 `.profile` 会 source） |
| `fish` | `~/.config/fish/config.fish` | |
| 其他 / 缺失 | `~/.profile` | POSIX 通用，两端登录 shell 都读 |

**一次 install 只改一个 rc 文件**（uninstall 只需清这一个）。

## uninstall 流程

1. 删 `~/.local/bin/llmw`（不存在则跳过）。
2. 按 marker 起始/结束行，从目标 rc 文件里**整段删除**那块（含边界行）。
   - 删除方式：纯 bash 逐行读 + 跳过块内行 + 写回，或 `awk`。**不用 `sed -i`**（见兼容性）。
3. 输出结果。

> uninstall 不读 `~/.local/bin` 是否在 `PATH` 做判断——只要 marker 块存在就删，幂等且
> 简单。

## 跨平台兼容

需覆盖 macOS、Ubuntu 及其他 Linux 发行版：

- **shebang**：`#!/usr/bin/env bash`，兼容 **macOS 自带 bash 3.2**——不使用 `mapfile` /
  `readarray` / 关联数组 / `**` 等 4+ 特性。
- **不用 `sed -i`**：BSD（macOS）与 GNU 的 `-i` 语义不兼容（macOS 需 `sed -i ''`，
  GNU 需 `sed -i`）。marker 块增删一律走**纯 bash 逐行读写**或 `awk`（两端行为一致）。
- **不假设 GNU coreutils**：`mkdir -p`、`chmod`、`rm` 等用 POSIX 基本用法；`uname` 判
  macOS。
- **PATH 命中检测**：POSIX `case ":$PATH:" in *":$HOME/.local/bin:"*)`。
- **shell / rc**：见上「rc 选择」。

## 错误处理

| 场景 | 行为 |
| --- | --- |
| `python3` 缺失 | 非 0 退出 + 提示安装 Python 3.7+ |
| Python < 3.11 | 提示需 `tomli`，继续安装（运行时才真正需要） |
| `~/.local/bin` 建目录 / 写 wrapper 失败 | 非 0 退出 + 报错 |
| rc 不可写 | 报错 + 提示手动添加 marker 块那一行 |
| 验证步骤（`--help`）失败 | 回滚已写产物 + 非 0 退出 + 报错 |
| 仓库被挪走（wrapper 内路径失效） | wrapper 运行时报友好错误，提示重跑 `./scripts/install.sh` |
| 重复 install | 幂等：覆盖 wrapper、不重复追加 marker 块 |
| 重复 uninstall | 幂等：wrapper 不存在则跳过、marker 不存在则跳过 |

**脚本退出码**：`0` = 成功，非 `0` = 失败（不复用 `llmw` CLI 的 1/2/3 语义，那是 Python
CLI 的约定）。

## 测试策略

### 自动 / 脚本化

- **install → 可用**：`./scripts/install.sh` 后 `~/.local/bin/llmw --help` exit 0。
- **幂等 install**：连跑两次，目标 rc 里 marker 块**只出现一次**；wrapper 被覆盖。
- **uninstall 干净**：uninstall 后 wrapper 不存在；目标 rc 里 marker 块**整段消失**；
  **块外行一字不变**（用 install 前后的 rc diff 校验，只有那一块被删）。
- **幂等 uninstall**：连跑两次，第二次无副作用、退出 0。
- **不改 rc 的路径**：当 `~/.local/bin` 已在 `PATH` 时 install 不写 rc；uninstall 时也无
  rc 改动。

### 手动（多平台 happy path）

- macOS（zsh，默认 bash 3.2）跑一遍 install → `llmw` 可用 → uninstall 干净。
- Ubuntu（bash）跑一遍同上。
